For my Session 12 assignment, I built a Comprehensive Study Planner. This isn’t just a simple list; it is a full-stack tool designed to help students organize their academic life. I used HTML for the structure, CSS for the styling, and Python (Flask) for the backend logic. All the data is stored in a data.json file, so the tasks stay saved even if I turn off my computer.
I am most proud of the interactive buttons I built: Delete, Deadline, Edit, and Check. However, the most important feature is the Priority selector. In my app, you don’t just write a task; you decide how Urgent and Important it is. This helps a user look at a long list and immediately know which task needs their attention right now and which one can wait.
The biggest technical challenge was the Edit button. It was difficult to make the app "remember" which task I wanted to change, pull that specific data back into the input box, and then save the update without creating a duplicate task. I overcame this by using unique IDs for every task and learning how to use the POST method in Flask to send that specific ID to my Python script.
Part 2: The Learning
The skill that improved the most during Phase 2 was my Backend Logic. Before this, I only knew how to make things look good with CSS. Now, I understand how to actually "talk" to a database. The concept that finally "clicked" was how JSON works as a middleman between my Python code and my HTML page.
I’m still a little confused by advanced CSS positioning, specifically making sure the Deadline and Priority badges stay perfectly aligned when the screen gets smaller on a mobile phone. To help with this, I used AI as a coding partner. AI was great at helping me find small syntax errors, like a missing bracket in my app.py, which saved me hours of frustration. It helped me understand why my code wasn't working rather than just giving me the answer.
Part 3: The Growth
Comparing "Session 7 me" to "Session 12 me," I have grown significantly. Six weeks ago, I could barely center a div. Now, I am writing Python functions that manage real data. I can now build a Persistent Data Application, which would have been impossible for me when we started.
What surprised me most about web development is how much logic goes into a simple Check button. When a user clicks "Check" to complete a task, the code has to find that specific entry in the data.json, change a "False" to a "True," and then refresh the styling to show a strike-through. It’s like a puzzle where every piece has to fit perfectly. Next, I want to learn JavaScript so I can make these updates happen instantly without the page reloading.
